# Agent 架构细化需求文档

> 基于原始需求，从 Agent 研发视角进行功能拆解与架构设计说明。

---

## 一、Agent 总览

| Agent 名称 | 角色定位 | 触发方式 |
|---|---|---|
| **Orchestrator Agent**（主控调度） | 系统入口，意图识别与任务路由 | 用户每次对话触发 |
| **QA Agent**（问答） | 知识问答与题目解答的核心执行者 | 由 Orchestrator 路由触发 |
| **Video Agent**（视频生成） | 负责完整的 manim 视频生成流水线 | 由 QA Agent 子任务触发 |
| **Knowledge Base Agent**（知识库管理） | 文档入库、解析、向量化、索引管理 | 管理员操作 / 独立离线触发 |
| **Learning Record Agent**（学情记录） | 对话后异步提取知识点、生成学情数据 | QA Agent 完成后异步触发 |

---

## 二、各 Agent 职责详述

### 1. Orchestrator Agent（主控调度 Agent）

**定位：** 系统统一入口，不直接处理业务逻辑，只做路由与编排。

**职责：**
- 接收用户原始输入（文本 / 图片 / 文件）
- 对输入做意图分类：
  - `qa`：知识点问答 / 题目解答
  - `video_request`：明确要求生成讲解视频
  - `knowledge_query`：查询知识库文档目录或内容
  - `learning_report`：查看个人学情反馈
- 将任务派发给对应 Agent，汇聚结果后统一返回给用户
- 管理整体对话上下文（短期记忆），负责多轮对话的上下文拼接

**输入：** 用户消息（含多模态附件）  
**输出：** 路由指令 + 最终聚合后的回复内容

---

### 2. QA Agent（问答 Agent）

**定位：** 核心业务 Agent，负责知识问答与解题。

**职责：**
- 解析用户问题（含图片 OCR 识别后的文字内容）
- 调用 **RAG Tool**，检索知识库中的相关文档片段作为上下文
- 基于检索上下文 + LLM 推理，生成结构化回答：
  - 纯文字解答
  - LaTeX 公式渲染
  - Markdown 格式输出
- 判断是否需要生成讲解视频（用户明确请求，或问题复杂度超过阈值）：
  - 若需要：将题目/知识点描述作为子任务派发给 **Video Agent**
  - 等待视频链接返回后，将链接附加到最终回答中
- 完成回答后，将本轮对话摘要（问题 + 涉及知识点）异步发送给 **Learning Record Agent**

**依赖工具（Tools）：**
- `rag_retrieval`：RAG 召回，返回相关文档片段
- `image_ocr`：处理用户上传的图片/扫描件，提取文字
- `file_parser`：解析用户上传的 PDF/MD 文件

**输入：** Orchestrator 路由的 `qa` 任务 + 用户上下文  
**输出：** 结构化回答文本（含可选的视频链接）

---

### 3. Video Agent（视频生成 Agent）

**定位：** 专职负责 manim 讲解视频的完整生成流水线，作为 QA Agent 的子 Agent 调用。

**职责（按流水线顺序）：**

1. **意图理解**：理解知识点或题目的核心内容，确定视频讲解角度
2. **分镜脚本生成**：
   - 按页（Scene）拆分内容块
   - 每页输出：文案描述、动画效果说明、旁白语音文案
3. **Manim 脚本生成**：根据分镜脚本生成可执行的 Python manim 代码
4. **Manim 渲染执行**（含重试机制）：
   - 调用 `manim_runner` 工具执行渲染
   - 若执行报错，捕获错误信息，自动修正脚本并重试，**最多重试 3 次**
   - 3 次后仍失败，返回失败状态与错误原因
5. **视频上传**：调用 `storage_upload` 工具将渲染产物上传至对象存储桶
6. **返回视频链接**：将最终可访问的视频 URL 返回给 QA Agent

**依赖工具（Tools）：**
- `manim_runner`：本地/远程执行 manim Python 脚本，返回渲染结果或错误信息
- `storage_upload`：上传视频文件至对象存储，返回访问链接

**输入：** 知识点描述 / 题目内容（来自 QA Agent）  
**输出：** 视频 URL 或失败原因

---

### 4. Knowledge Base Agent（知识库管理 Agent）

**定位：** 离线/后台 Agent，负责文档的接入、解析与向量化，为 RAG 提供索引基础。（非实时对话链路，通常由管理员操作触发）

**职责：**
- **文档接收与分类**：按科目（英语/政治/高数等）维护树状目录结构，管理文档路径与元数据
- **文档解析 Pipeline**：
  - `.md` 文件：直接按标题/段落结构化解析
  - `.pdf` 文件（含文字）：提取文本内容
  - `.pdf` 文件（全图片型）：调用 OCR 工具提取图片中的文字
- **Chunk 处理**：
  - 按语义单元进行合理切分
  - 为每个 chunk 附加清晰的 metadata（科目、文档路径、章节、页码等）
- **向量化与索引**：将 chunk 向量化后写入向量数据库
- **索引更新**：支持文档增删改时的增量索引更新

**依赖工具（Tools）：**
- `pdf_parser`：PDF 文本提取
- `pdf_ocr`：图片型 PDF 的 OCR 识别
- `text_chunker`：文本切分
- `vector_embedder`：文本向量化
- `vector_db_writer`：写入向量数据库

**输入：** 上传的文档文件 + 科目分类信息  
**输出：** 向量数据库中的索引记录

---

### 5. Learning Record Agent（学情记录 Agent）

**定位：** 异步后台 Agent，无感知地运行在每次 QA 完成之后，不参与实时对话链路。

**职责：**
- 接收 QA Agent 传来的对话摘要（用户问题 + 解答内容）
- 提取本次对话涉及的**知识点标签**（科目 → 章节 → 知识点，结构化）
- 将知识点记录持久化到数据库（含频次、时间戳、正确率等维度）
- 定期或按需汇总用户的学情数据，生成**学情反馈报告**：
  - 常问知识点 TOP N
  - 薄弱知识点分析
  - 学习趋势（时间维度）

**依赖工具（Tools）：**
- `knowledge_point_extractor`：从对话文本中提取结构化知识点标签
- `db_writer`：将学情记录写入关系型数据库
- `report_generator`：根据历史数据生成学情报告

**输入：** QA Agent 发出的对话摘要事件（异步消息）  
**输出：** 结构化学情记录（写入 DB），以及按需返回的学情报告

## 三、Agent 关系与调用链路

### 3.1 主流程（用户发起问答）

```
用户输入
    │
    ▼
Orchestrator Agent
（意图识别 → 路由）
    │
    ▼ [qa / video_request]
QA Agent
    ├─── RAG Tool ──→ 知识库向量检索 ──→ 返回上下文片段
    ├─── [需要视频时] ──→ Video Agent（子任务，同步等待）
    │                        ├─ 分镜脚本生成
    │                        ├─ Manim 脚本生成
    │                        ├─ Manim 渲染（含重试 ≤3次）
    │                        └─ 上传存储桶 → 返回视频 URL
    │
    ├─── 返回最终回答（文字 + 可选视频链接）
    │
    └─── [异步] ──→ Learning Record Agent
                       └─ 提取知识点 → 写入 DB
```

### 3.2 知识库管理流程（离线）

```
管理员上传文档
    │
    ▼
Knowledge Base Agent
    ├─ 文档解析（md / pdf文字 / pdf图片→OCR）
    ├─ Chunk 切分 + Metadata 标注
    └─ 向量化 → 写入向量数据库
              （供 QA Agent 的 RAG Tool 检索使用）
```

### 3.3 学情查询流程

```
用户请求学情报告
    │
    ▼
Orchestrator Agent
（意图：learning_report）
    │
    ▼
Learning Record Agent
    └─ 查询 DB → 生成报告 → 返回
```

## 四、Agent 间通信方式

| 通信场景 | 方式 | 说明 |
|---|---|---|
| Orchestrator → QA Agent | 同步调用 | 等待回答后统一返回用户 |
| QA Agent → Video Agent | 同步子任务调用 | QA Agent 等待视频链接后再返回完整回答 |
| QA Agent → Learning Record Agent | 异步事件消息 | 对话完成后 fire-and-forget，不阻塞主链路 |
| 管理员 → Knowledge Base Agent | 独立触发 | 与对话链路完全解耦，离线执行 |

---

## 五、记忆系统归属

| 记忆类型 | 归属 Agent | 说明 |
|---|---|---|
| 短期记忆（对话上下文） | Orchestrator Agent | 维护当前会话的多轮上下文窗口 |
| 向量记忆（语义检索） | QA Agent via RAG Tool | 知识库向量索引，由 KB Agent 维护 |
| 结构化长期记忆（学情） | Learning Record Agent | 用户历史问答的知识点数据库记录 |

---

## 六、补充说明

- **多科目支持**：QA Agent 的 RAG 检索会根据问题自动识别所属科目，过滤对应科目的文档进行检索，避免跨科目噪声干扰。
- **图片/文件输入**：用户上传的图片由 QA Agent 的 `image_ocr` 工具预处理，再进入正常问答流程；用户上传的 PDF 文件同理。
- **Video Agent 的独立性**：Video Agent 完全封装视频生成细节，QA Agent 只需传入知识点描述，不感知视频生成内部逻辑，便于后续替换渲染引擎或增加其他视频后处理步骤。
- **Knowledge Base Agent 的离线性**：KB Agent 不在实时对话链路中，文档更新不会影响在线问答服务的稳定性。
