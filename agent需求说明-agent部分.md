## 需求概述
我需要做一个agent的系统，大致是为大学生/考研人服务的学习辅助平台，提供问答服务，具体核心功能点说明如下：

> 目前计划仅支持英语、政治、高等数学、线性代数、概率论与数理统计、计算机组成原理、操作系统、数据结构、计算机网络
>

+ 知识点问答/题目解答
    - 输入：
        * 文字解答
        * 图片/文件上传
    - 输出：
        * 文字解答
        * 支持latex格式展示
        * md展示
        * 讲解视频生成（基于manim，后面细说）
+ 知识库
    - 相关文档会按科目进行分类，文档支持md、pdf格式，而pdf里可能全是图片，需要考虑怎么处理（不同文档存储路径是树状关系）
    - RAG也是需要的
+ 学习记录
    - 根据用户的对话，记录用户的常问内容涉及到的知识点并记录成结构化数据，用于生成学情反馈

**【视频生成大致流程说明】**

+ 用户意图识别（知识点解析/问题解答）
+ 生成分镜脚本
    - 按页分块
    - 当前页文案、动画效果描述
    - 语音文案内容说明
+ 生成对应manim脚本
+ manim渲染成视频（如果脚本报错，需要有根据报错信息的重试机制，最多三次）
+ 视频上传至存储桶
+ 返回视频链接

# 技术说明
+ 提示词工程
    - 结构定义：角色定义、任务描述、上下文信息、输出要求
    - 结构模板化
    - Prompt参数化
    - Prompt分层设计
+ 输出要求
    - 结构化输出
    - Schema约束，以yaml的形式而不是json，以免解析报错
+ RAG
    - Query输入
    - Query向量化
    - 向量检索
    - 上下文构建
    - 向量数据库构建，选择合适的向量数据库、检索算法
    - 文档解析
    - Chunk处理
        * <u>构建稳定的文档解析 Pipeline</u>
        * <u>合理的 Chunk 切分策略</u>
        * <u>清晰的 Metadata 结构</u>
    - RAG召回优化
    - Rerank
    - 上下文压缩
    - Multi-source RAG（可能涉及数据库数据、文档内容两种不同来源）
+ Agent本身设计（Harness工程化）
    - LLM Reasoning 推理
        * 推理模式设计
    - Tool Selection
    - Tool Execution
    - Function Calling
    - Skill设计
        * 单一职责
        * 高内聚、低耦合
        * 描述清晰准确
        * 输入输出结构化
        * 可组合
    - 规则层
    - 记忆系统
        * 架构设计
            + 上下文管理
            + 向量记忆（针对语义的记忆）
            + 数据库记忆（针对结构化的数据）
        * 分类
            + 短期记忆
            + 长期记忆
    - 反馈与验证机制/重试机制
    - 可观测性
        * 日志记录
        * 链路追踪
        * 系统监控
    - 成本控制
        * Token优化
        * 缓存策略
        * 小模型策略（对于简单任务）
    - 稳定性设计
        * 重试机制
        * 安全防护（例如用户的输入）
        * 降级策略
+ multi-agent架构设计（根据我的需求看是否是multi-agent）
    - 架构设计
    - 通信方式
    - 协作模式
+ agent自动测评
    - 成功率
    - 延迟
    - 成本
    - 综合计算根据以上三个指标
